Richard Lamy

Cybersecurity Apprentice

Hello — I'm Richard Lamy, a cybersecurity apprentice and second-year student at Teesside University. I combine academic study with hands-on experience to design and implement practical security solutions, and I am particularly focused on penetration testing and risk assessment. I am driven by curiosity, continual learning and a commitment to helping organisations improve their security posture.

My Ambitions

I am committed to specialising in ethical hacking and apply structured methodologies such as the OWASP testing framework to identify and remediate vulnerabilities while strictly observing legal and ethical boundaries. My experience as a basketball team leader — making rapid, high-stakes decisions while preserving team cohesion and managing interpersonal dynamics — has developed my situational awareness, decisive judgement and conflict-sensitive communication, all of which transfer directly to collaborative security engagements and incident response.

I conduct penetration tests only within authorised scopes, maintain clear rules of engagement, preserve chains of evidence, and report findings responsibly to minimise organisational risk and protect stakeholder trust. A dedicated researcher and reader, I pursue interests ranging from contemporary politics to classical Greek and Norse strategic thought; this intellectual curiosity fuels my analytical rigour, pattern recognition and methodical approach to investigation.

In the long term I aim to progress into specialist roles such as forensic ethical hacker — performing incident investigations, evidence preservation and attribution analysis — and to serve in high-impact environments, whether within government cyber units (for example national CERTs or allied organisations), defence/armed-forces cyber teams, or within the security teams of regulated financial institutions. To reach these goals I intend to continue developing technical depth, pursue relevant certifications, and cultivate the professional ethics and operational discipline required to work effectively in regulated and mission-critical contexts.

Technical Skills

I possess a solid and versatile technical foundation that combines analytical thinking with practical expertise across key areas of information technology and cybersecurity.
Programming
60%▶
I write reliable, maintainable scripts and small applications using Python, JavaScript and Bash. I apply software engineering best practice: modular design, version control with Git, unit testing and simple CI, and use static analysis tools (e.g. Bandit, ESLint) to catch common issues early. I build automation for security tasks (scanning, parsing logs, report generation), prototype PoCs for vulnerability analysis, and use libraries such as Requests, BeautifulSoup and Pandas for data collection and analysis. I pay attention to secure coding—input validation, safe deserialisation, proper error handling and secure dependency management—to reduce attack surface and ease forensic investigation.
Cyber Security Risk Assessment
70%▶
I perform structured risk assessments aligned to recognised frameworks (NIST SP 800-30, ISO 27001) and use threat modelling techniques (STRIDE, attack trees) to identify likely threat scenarios. My process includes asset discovery, vulnerability scanning (Nmap, Nessus/OpenVAS), CVSS-based risk scoring, control-gap analysis and prioritised mitigation planning. I document rules of engagement, evidence chains and residual risk statements, and produce actionable remediation roadmaps (patching, configuration changes, compensating controls). I also design and recommend continuous controls — logging, alerting, MFA and segmentation — so risk reduction is measurable and auditable.
SQLite / SQL Basics
50%▶
I work with relational data using SQL for querying, aggregation, joins and basic schema design, typically in SQLite for lightweight tooling and SQL Server/MySQL for larger datasets. I write parameterised queries to prevent SQL injection, use transactions to preserve ACID properties, and apply indexing and query-explain techniques to improve performance. I extract and normalise forensic artefacts (audit logs, application traces) to support incident analysis and can script ETL tasks to transform raw logs into evidence-ready formats. I'm familiar with basic backup strategies and simple database encryption options for protecting data at rest.
Website (HTML & CSS)
60%▶
I develop semantically structured, responsive front-ends using HTML5 and modern CSS (Flexbox, Grid) with progressive enhancement in mind. Accessibility (WCAG) and security are core considerations: ARIA attributes, keyboard navigability, Content Security Policy, secure headers and input sanitisation to mitigate XSS/CSRF. I optimise for performance using lazy-loading, asset minification and Lighthouse-driven improvements, and I validate designs across desktop and mobile viewports. For interactive features I use unobtrusive JavaScript and follow secure API consumption patterns (CSRF tokens, CORS restrictions and token-based auth).
Networking / Packet Tracer
70%▶
I design and simulate network topologies in Cisco Packet Tracer, covering IP addressing and subnetting, VLANs, inter-VLAN routing, static routes and dynamic routing protocols (OSPF basics). I configure ACLs, NAT, and basic QoS, and perform packet-level analysis to debug connectivity with packet captures (Wireshark). I apply segmentation and least-privilege principles to limit lateral movement, model attacker paths to identify choke-points, and prepare configuration hardening checklists (disable unused services, secure management interfaces, change default SNMP community strings). These simulations are used to validate secure designs before deployment and to rehearse incident responses.
